이벤트 처리
위 이벤트가 발생했을 때 뭘 하고 싶은지 물음

이벤트가 발생하는 곳, 이벤트 처리하는 곳은 다를 수 있다.

keydown		onkeydown	키가 눌렸을 때
click		onclick		클릭했을 때
doubleclick	ondoubleclick	더블클릭했을 때
mouseover	onmouseover	마우스를 올렸을 때
mouseleave	onmouseleave	마우스가 벗어났을 때
change		onchange	요소의 값이 변경(포커스가 사라졌을 때)
input		oninput		사용자가 요소에 값을 입력할 때
submit		onsubmit		form요소의 제출버튼이 클릭되었을 때
reset		onreset		form요소의 리셋버튼이 클릭되었을 때


인라인 이벤트 모델
<p onclick ="이벤트 처리">

표준 이벤트 모델
<p> </p>
<Script>
이벤트 처리 대상을 정한다.
let p = document.querySelector("p")
</script>

지킬 생태계
생태계라고 부를만큼 다양한 사용자들이 존재한다.
덕분에 우리는 누군가 만들어 놓은 템플릿을 열람할 수 있다.

Agency템플릿 수집하기
http://jekyllthemes.org/themes/agency

마음에 드는 템플릿을 찾았으면 라이센스 확인하기

mit라이센스 
이 소프트웨어를 누구든지 무상으로 제한없이 사용해도 좋다.
단,완성된 소프트웨어의 중요한 부분에 저작권을 표시해야한다.
소프트웨어의 원저작자는 아무런 책임을 지지 않는다.

레포지토리 포크
오래된 템플릿은 소개 사이트에는 있어도,원본코드가 유실되어 사용이 곤란한 경우가 있다.
따라서 템플릿의 깃허브 레포지토리가 살아 있는지 확인해야 한다.

템플릿의 이해
내가 사용하려고 하는 템플릿의 구조를 이해해야 한다.

포트폴리오 탭에서 포트폴리오 사진을 하나 누르게 되면
모달이라고 부르는 창이 뜬다.
자바스크립트를 이용해 직접 만들면 굉장히 골치 아프고 복잡한데 이런 유용한 기능이 기본적으로 제공되고 있다.

jekyll serve 명렁어 실행시 에러나면 다음의 명령어를 순차적으로 실행

bundle add webrick

지킬 프로젝트 기본 구조

_config.yml 	웹페이지의 환경설정과 정보를 보관
_data/		웹페이지에서 활용할 데이터를 저장
_includes/	-웹페이지에 포함 시킬 html 파일
		-주로 개별 페이지들이나 단락 저장
_layouts/		웹 페이지의 뼈대를 결정
_post/		블로그의 게시물을 저장
_site/		CSS의 변종인 sass(scss)파일들을 저장
_sass/		-index.html 역할을 수행할 파일 저장
		-혹은 규격에 맞춘 마크다운 파일도 사용 가능


지킬 템플릿 수정하기
_config.yml
거의 모든 지킬 템플릿은 _config.yml이라는 파일을 포함하고 있다.
웹페이지의 메타데이터를 수정할 수 있도록 제작되어 있다.
