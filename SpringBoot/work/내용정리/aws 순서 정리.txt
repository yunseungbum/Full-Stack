aws 순서 정리
1. 배포하기 위한 프로그램을 만든다.

2. 프로그램을 빌드한다.
  2-1 스프링부트의 경우 gradlew가 있는 폴더로 경로를 잡고 ./gradlew build를 한다.
  2-2 리엑트의 경우 npm run build를 한다.(설정파일과 묶어서 압축한다.)

3. Elastic Beanstalk 환경을 만든다.
  3-1 백엔드 환경을 만든다.
	3-1-1 빌드된 .jar 파일을 배포한다.
  3-2 프론트엔드 환경을 만든다.
	3-2-1 빌드된 압축파일을 배포한다.

4.오류나면 로그를 확인한다.

5.코드를 수정한다. -> 각 환경으로 다시 배포를 한다.
